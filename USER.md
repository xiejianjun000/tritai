## 十、商业思维特征（2026年4月16日补充）

### 8.1 架构化定价思维
用户习惯于**先搭架构，再定价格**，而不是先报价再设计：
- 定价顺序：设计架构 → 分层定价 → 报价（不是客户问价→报价→设计）
- 报价方式：分项报价（平台费+授权费+SKILL费），不是一口价
- 溢价来源：架构溢价（复用降成本），而不是关系溢价

### 8.2 复利思维
用户所有的设计都指向**持续收入**，而不是一次性交易：
```
一次性收入：平台开发费 50-150万
     ↓
持续收入：年授权费 10-30万/年
     ↓
生态收入：SKILL抽成 30%
     ↓
指数增长：开发者越多 → SKILL越多 → 用户越多 → 收入越多
```

### 8.3 激励对齐思维
用户总是设计"各方受益"的机制，而不是"我赢你输"：
- 开发者：70%分成，愿意开发更多SKILL
- 用户：能力提升，愿意购买更多SKILL
- 租户：效率提升，愿意续费
- 平台：抽成+授权费，愿意持续运营

### 8.4 场景优先思维
用户总是从**用户场景出发**，而不是从技术出发：
- 错误路径：技术（Hermes）→ 找应用场景 → 卖技术
- 正确路径：用户场景（环评审查）→ 能力需求 → 技术选型 → 架构设计 → 定价
- 今天体现：
  - "用户如何知道要用SKILL？" → 獬豸引流
  - "买了之后在哪里用？" → OpenClaw门户
  - "Hermes如何知道权限？" → OpenClaw验证

### 8.5 思维演进特征
用户在讨论中展现出清晰的思维跃迁：
- 从技术导向 → 架构分离 → 复制策略 → 生态布局 → 用户旅程
- 每个阶段都能快速理解矛盾本质，提出解决方案
- 善于通过提问引导深度思考

### 8.6 核心竞争力
- 架构能力：识别分层、设计复用
- 商业嗅觉：发现盈利点、设计定价
- 生态思维：激励对齐、各方受益
- 技术理解：知道什么能落地

### 8.7 思维盲区（自我认知）
