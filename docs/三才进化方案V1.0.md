## 一、架构设计

```
┌─────────────────────────────────────────────────────────┐
│                    三才进化闭环                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  用户反馈 ──→ 反馈收集器 ──→ 梦境分析 ──→ 规则进化      │
│     ↓            ↓              ↓              ↓        │
│  误报标记    FeedbackStore   DreamingSystem  Evolution  │
```

## 二、分层进化策略

### L1 核心规则层（锁定，不进化）
```javascript
const LOCKED_RULES = ['R001', 'R003', 'R004', 'R005', 'R006'];
// 这些规则不可被自动修改，保证基础稳定性
```

### L2 用户偏好层（个人进化）
- 阈值调整（用户标记误报后降低敏感度）
- 置信度权重（根据用户场景动态调整）
- 白名单/黑名单（用户自定义例外）

### L3 领域知识层（订阅更新）
- 法律知识包（由内容商控制）
- 金融知识包
- 医疗知识包
- 环保知识包

## 三、反馈收集器

### 3.1 反馈类型
```javascript
enum FeedbackType {
  FALSE_POSITIVE = 'false_positive',  // 误报：正常文本被误判为幻觉
  FALSE_NEGATIVE = 'false_negative',  // 漏报：幻觉文本未被检测
  KNOWLEDGE_CORRECTION = 'knowledge_correction',  // 知识纠正
  USER_PREFERENCE = 'user_preference'  // 用户偏好设置
}

interface Feedback {
  id: string;
  type: FeedbackType;
  text: string;  // 原始文本
  result: any;   // 原始检测结果
  userComment?: string;  // 用户备注
  correctedFact?: string;  // 正确的事实
  timestamp: string;
  userId: string;
  context?: {
    domain?: string;  // 领域：法律/金融/医疗/环保
    scenario?: string;  // 场景：报告/合同/诊断
  };
}
```

### 3.2 存储为记忆条目
```javascript
async function recordFeedback(feedback: Feedback) {
  const memoryEntry = {
    id: feedback.id,
    content: JSON.stringify(feedback),
    type: 'feedback',
    metadata: {
      feedbackType: feedback.type,
      domain: feedback.context?.domain,
      userId: feedback.userId,
      importance: feedback.type === 'FALSE_NEGATIVE' ? 0.9 : 0.7
    },
    timestamp: new Date().toISOString()
  };
  
  // 存储到太极开源记忆系统
  await taijiClient.storeMemory(memoryEntry);
}
```

### 4.3 梦境结果解析
```javascript
function extractEvolutionPlan(dreamResult: DreamResult) {
  const plan = {
    // 从洞见提取新规则建议
    newRules: [],
    // 从矛盾修复提取调整建议
    ruleAdjustments: [],
    // 从图谱整合提取知识更新
    knowledgeUpdates: []
  };
  
  // Phase3: 洞见提取
  for (const insight of dreamResult.insights || []) {
    if (insight.type === 'pattern' && insight.confidence > 0.8) {
      // 发现新的幻觉模式，生成候选规则
      plan.newRules.push({
        source: insight.content,
        evidence: insight.evidence,
        confidence: insight.confidence,
        status: 'candidate'  // 需人工确认
      });
    }
  }
  
  // Phase4: 矛盾修复
  for (const repair of dreamResult.repairs || []) {
    if (repair.type === 'update') {
      plan.ruleAdjustments.push({
        targetRuleId: repair.targetMemoryId,
        action: 'adjust_threshold',
        reason: repair.reason,
        priority: repair.priority
      });
    }
  }
  
  // Phase5: 图谱整合
  const integration = dreamResult.integration;
  if (integration) {
    plan.knowledgeUpdates = {
      newClaims: integration.claims,
      newEntities: integration.entities,
      newRelations: integration.relations
    };
  }
  
  return plan;
}
```

## 五、进化应用策略

### 5.1 分层应用
```javascript
async function applyEvolution(plan) {
  const results = {
    rulesCreated: 0,
    rulesAdjusted: 0,
    knowledgeUpdated: 0,
    rejected: []
  };
  
  // 1. 新规则创建（候选状态，需确认）
  for (const newRule of plan.newRules) {
    // 不直接创建，存入候选池
    await saveCandidateRule(newRule);
    results.rulesCreated++;
  }
  
  // 2. 规则调整（分层策略）
  for (const adjustment of plan.ruleAdjustments) {
    const rule = getRule(adjustment.targetRuleId);
    
    // L1核心规则不调整
    if (LOCKED_RULES.includes(rule.id)) {
      results.rejected.push({
        reason: '核心规则不可调整',
        ruleId: rule.id
      });
      continue;
    }
    
    // L2用户偏好层可调整
    await adjustUserPreference(rule, adjustment);
    results.rulesAdjusted++;
  }
  
  // 3. 知识更新
  if (plan.knowledgeUpdates) {
    await taijiClient.injectKnowledge(plan.knowledgeUpdates);
    results.knowledgeUpdated = plan.knowledgeUpdates.newClaims.length;
  }
  
  return results;
}
```

### 5.2 用户偏好存储
```javascript
// 用户偏好独立存储，不影响其他用户
async function adjustUserPreference(rule, adjustment) {
  const preference = {
    userId: adjustment.userId,
    ruleId: rule.id,
    thresholdAdjustment: adjustment.thresholdDelta,
    reason: adjustment.reason,
    timestamp: new Date().toISOString()
  };
  
  await saveUserPreference(preference);
}

// 验证时合并用户偏好
function verifyWithUserPreferences(text, userId) {
  const preferences = getUserPreferences(userId);
  const result = wfgyEngine.verify(text);
  
  // 应用用户偏好调整
  for (const pref of preferences) {
    const rule = result.matchedRules.find(r => r.id === pref.ruleId);
    if (rule) {
      rule.confidence *= (1 + pref.thresholdAdjustment);
    }
  }
  
  return result;
}
```

## 八、安全控制

### 8.1 进化限制
- 核心规则不可自动修改
- 新规则需人工确认
- 用户偏好仅影响该用户

### 8.2 回滚机制
```javascript
// 每次进化前快照
async function createSnapshot() {
  return {
    rules: cloneDeep(wfgyEngine.rules),
    knowledgeBase: cloneDeep(wfgyEngine.knowledgeBase),
    timestamp: new Date().toISOString()
  };
}

// 回滚到快照
async function rollback(snapshot) {
  wfgyEngine.rules = snapshot.rules;
  wfgyEngine.knowledgeBase = snapshot.knowledgeBase;
}
```

### 8.3 审计日志
```javascript
// 记录所有进化操作
async function auditLog(action, details) {
  await logAudit({
    module: 'TriTai-Evolution',
    action,
    details,
    timestamp: new Date().toISOString()
  });
}
```
