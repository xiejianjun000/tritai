# WFGY 防幻觉引擎核心原理

## ⚠️ 核心定位

**WFGY是三才的技术核心**，其他一切能力都为它加持：
- 知识图谱 → 为WFGY提供事实依据
- 进化闭环 → 让WFGY规则自我优化  
- 订阅领域包 → 为WFGY注入领域知识

## 名称由来

**WFGY = Well-Founded Goodness Yielding**
- **Well-Founded**: 基于确定性规则，有据可依
- **Goodness**: 产生高质量检测结果
- **Yielding**: 零Token消耗，纯符号计算产出

## 符号层原理

### 核心思想
用**符号规则**匹配文本模式，判断是否为AI幻觉。

### 符号结构

```javascript
{
  id: 'R003',                    // 符号标识
  name: '时间穿越',              // 符号名称
  pattern: /20[2-9]\d年/g,       // 正则匹配模式
  type: 'future_hallucination',  // 幻觉类型
  baseConfidence: 0.90,          // 基础置信度
  
  // 验证函数：判断是否真的是幻觉
  check: (text, match) => {
    const year = parseInt(match);
    return year > new Date().getFullYear();
  },
  
  // 纠正函数：生成纠正建议
  correction: (match, text) => {
    return `"${match}"是未来时间，无法作为已发生事件的依据`;
  }
}
```

### 验证流程

```
输入文本
    ↓
符号匹配（正则pattern）
    ↓
验证函数判断（check）
    ↓
置信度计算
    ↓
生成纠正建议（correction）
    ↓
输出：{ detected, confidence, evidence, correction }
```

## 置信度算法

```javascript
// 单条规则触发
confidence = rule.baseConfidence

// 多条规则触发（叠加）
confidence = max(confidence) + 0.05 * count
// 上限 0.98
```

## 为什么是核心？

### 1. 零Token消耗
纯符号计算，不调用LLM，无额外成本。

### 2. 毫秒级响应
正则匹配 + 简单逻辑判断，速度极快。

### 3. 可解释性强
每条规则都是明确的符号逻辑，可追溯可理解。

### 4. 持续进化
规则可动态添加、修改、淘汰，持续优化。

### 5. 领域可定制
通过订阅包注入领域专属规则。

## 开发指南

### 添加新规则

```javascript
// 1. 定义规则符号
const newRule = {
  id: 'R007',
  name: '数字幻觉',
  pattern: /\d+(\.\d+)?/gi,
  type: 'numeric_hallucination',
  baseConfidence: 0.85,
  check: (text, match) => {
    // 自定义验证逻辑
    return parseFloat(match) > 1000000;
  },
  correction: (match) => {
    return `"${match}"数值异常，请核实`;
  }
};

// 2. 注入到引擎
wfgyEngine.addRule(newRule);

// 3. 测试验证
const result = wfgyEngine.detect("光速是300万km/s");
console.log(result);
```

### 调用验证

```javascript
// 基础验证
const result = wfgyEngine.detect(text);

// 增强验证（+知识图谱）
const result = await wfgyEngine.verifyWithKnowledgeGraph(text);
```

## 太极阴阳哲学

### ☀️ 阳：确定性 — WFGY防幻觉系统

**阳是规则，是秩序，是可验证的确定性**

WFGY是三才的「阳」——提供确定性边界：

```
┌───────────────────────────────────────────────────────────────────┐
│                      WFGY 防幻觉五重验证                          │
├────────────────┬───────────────────┬──────────────┬───────────────┤
│  WFGYVerifier  │ SelfConsistency   │ SourceTracer │ Hallucination │
│  符号层规则验证  │ 多路径自一致性检查  │ 知识溯源索引  │ 幻觉风险检测器  │
└────────────────┴───────────────────┴──────────────┴───────────────┘
```

- **符号层规则验证**：基于知识库的事实匹配，输出必须在知识边界内
- **多路径自一致性**：对同一问题采样多次，投票选出一致结果
- **知识溯源索引**：每个结论都能追溯到原始知识来源，可审计
- **幻觉风险检测**：实时计算输出的置信度，标记高风险内容

### 🌙 阴：随机性 — LLM创造力引擎

**阴是变化，是创造，是不可预测的可能性**

LLM是三才的「阴」——提供创造力源泉：

- 生成新知识
- 推理复杂问题
- 创造性表达

### ⚖️ 阴阳调和：动态平衡机制

**核心创新**：不是让确定性去「限制」随机性，而是让两者**对话**。

```
LLM输出（阴）
    ↓
[WFGY检测器打分] → 分数 < 阈值？
    ↓                  ↓
  直接输出         [自一致性检查] → 3次采样投票
                          ↓
                    [溯源索引] → 匹配知识库
                          ↓
                    输出 + 来源标注 + 置信度
```

**阳中有阴，阴中有阳。这就是太极。**

### 阴阳平衡原则

1. **阳不压阴**：规则不过度限制创造力
2. **阴不离阳**：创造不脱离事实边界
3. **动态平衡**：根据场景调整阴阳比例
4. **螺旋上升**：阴中有阳的进化，阳中有阴的适应

---

**一句话总结**：WFGY是阳，给LLM的阴提供确定性的容器。阴阳调和，方为太极。
