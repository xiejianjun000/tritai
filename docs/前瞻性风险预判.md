# 三才前瞻性风险预判与应对

## 一、规则层风险

### 风险1：规则冲突
**场景**：新规则R007与老规则R003对同一段文本给出矛盾判断
**应对**：
- 规则优先级系统（weight字段）
- 冲突检测机制（Phase4已实现）
- 规则依赖声明（rule.dependencies）

### 风险2：规则过时
**场景**：2027年发布新法规，规则还在检测2026年的错误说法
**应对**：
- 规则有效期字段（validFrom, validUntil）
- 法规变更监控订阅
- 自动失效提醒

### 风险3：规则通胀
**场景**：进化系统持续添加规则，规则库膨胀影响性能
**应对**：
- 规则合并机制（相似规则合并）
- 低效规则淘汰（命中率<阈值自动下线）
- 规则分层存储（热规则内存，冷规则磁盘）

## 二、知识层风险

### 风险4：知识过时
**场景**：生态环境法典实施细则发布，知识包没更新
**应对**：
- 知识包版本号（semantic versioning）
- 法规变更订阅源（官方API接入）
- 用户订阅到期提醒+知识包更新通知

### 风险5：知识污染
**场景**：恶意用户批量提交错误反馈污染知识库
**应对**：
- 反馈可信度评估（用户等级、历史准确率）
- 反馈权重衰减（时间越久权重越低）
- 知识变更审计日志
- 关键知识锁定机制

### 风险6：领域漂移
**场景**：法律用户和医疗用户对同一规则的敏感度需求不同
**应对**：
- 用户画像系统（领域标签、场景标签）
- 领域自适应阈值
- 用户级规则覆盖（user_overrides）

## 三、进化层风险

### 风险7：过度进化
**场景**：系统为追求"高准确率"把所有可疑文本都判为幻觉
**应对**：
- 进化目标函数平衡（precision + recall + F1）
- 进化速率限制（每次最多调整N%）
- 进化效果监控（A/B测试）

### 风险8：进化震荡
**场景**：规则A今天调高明天调低，反复震荡
**应对**：
- 进化动量机制（调整方向一致性加权）
- 冷却期（同一规则两次调整间隔≥N小时）
- 快照回滚（震荡超过阈值自动回滚）

### 风险9：进化不透明
**场景**：用户不知道为什么规则变了，失去信任
**应对**：
- 进化仪表盘（可视化展示进化历史）
- 变更通知机制（规则调整后通知订阅用户）
- 解释生成（每次进化生成人类可读说明）

## 四、订阅层风险

### 风险10：续费断崖
**场景**：用户订阅到期后不再续费，知识包停更
**应对**：
- 订阅价值仪表盘（展示检测次数、拦截幻觉数）
- 到期前7天提醒+优惠续费
- 免费版能力限制（倒逼付费）

### 风险11：知识包版本混乱
**场景**：用户A订阅v1.0，用户B订阅v1.2，检测结果不一致
**应对**：
- 知识包版本锁定（订阅时绑定版本）
- 强制升级机制（安全修复强制推送）
- 版本兼容性声明（v1.x兼容v1.y）

### 风险12：跨领域依赖
**场景**：环保知识包依赖法律知识包，用户只订阅环保
**应对**：
- 依赖声明（knowledge_pack.dependencies）
- 依赖打包销售（环保+法律套餐）
- 依赖降级处理（缺失依赖时降级检测）

## 五、系统层风险

### 风险13：单点故障
**场景**：太极开源服务挂了，三才完全不可用
**应对**：
- 本地缓存（知识图谱快照本地备份）
- 降级模式（纯本地WFGY检测）
- 多节点部署（负载均衡+故障转移）

### 风险14：性能瓶颈
**场景**：用户量暴增，梦境系统处理不过来
**应对**：
- 进化队列（反馈先进队列，异步处理）
- 批量梦境（攒够N条反馈再触发）
- 进化优先级（漏报优先级>误报）

### 风险15：数据泄露
**场景**：用户反馈的敏感文本被泄露
**应对**：
- 反馈脱敏（敏感词替换）
- 数据加密存储
- 访问审计
