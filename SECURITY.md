# 安全政策

## 报告安全漏洞

如果您发现了安全漏洞，请**不要**使用公开的 Issue 报告。请通过以下方式私下联系维护者：

### 报告渠道

- **GitHub Security Advisories**：建议使用此方式报告，您可以在 [Security](https://github.com/xiejianjun000/tritai/security) 页面提交漏洞报告
- **邮件**：发送邮件至 [INSERT_EMAIL_HERE]（请加密敏感信息）

### 报告内容

请在报告中包含以下信息：

1. **漏洞描述** - 清晰描述漏洞的性质和影响
2. **复现步骤** - 详细描述如何复现该漏洞
3. **受影响版本** - 哪些版本受到影响
4. **可能的修复方案** - 如果您有修复建议，请说明
5. **您的信息** - 您的姓名和联系方式（可选）

### 响应时间

- 我们将在 **48 小时内**确认收到您的报告
- 我们将在 **7 天内**提供漏洞的初步评估和处理计划
- 对于严重漏洞，我们将优先处理并尽快发布修复

## 安全最佳实践

### 用户

- 始终使用最新的稳定版本
- 定期检查安全更新
- 不要在公共代码仓库中提交敏感信息（API 密钥、密码等）
- 使用安全的认证方式
- 定期轮换访问凭证

### 开发者

- 遵循安全编码规范
- 对所有用户输入进行验证和清理
- 使用参数化查询防止 SQL 注入
- 实施适当的身份验证和授权
- 对敏感数据进行加密
- 定期进行安全代码审查

### 依赖管理

- 使用 Dependabot 自动检测依赖漏洞
- 定期更新依赖包到最新安全版本
- 检查依赖包的安全历史

## 支持的版本

我们只为以下版本提供安全更新：

| 版本 | 支持状态 |
|------|----------|
| 最新版 | ✅ 支持 |
| 旧版 | ❌ 不支持 |
